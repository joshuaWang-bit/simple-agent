# 小结与展望

s4 没有新增 agent 的"能力"，但彻底改变了它的"存在形态"。

| 维度 | s3 | s4 |
|------|-----|-----|
| 运行单位 | 孤立的 run | session 里的一个回合 |
| 记忆 | 无 | thread（对话历史）+ notes（事实笔记） |
| 客户端 | 一次性命令 | 持续会话（chat / TUI） |
| 用户交互 | 发完就等 | 多轮输入，边等边看 |

session 不是简单的"把几次 run 拼在一起"。它引入了两层存储、四条读写路径，以及一个核心约束：**用户消息必须先写 thread，再启动 run**。这个顺序保证了 LLM 看到的上下文和用户实际输入完全一致。

`note_save` 是 s4 埋下的一个伏笔。短对话里它看起来多余——thread 已经够用了。但当 s6 引入 compact 后，notes 会成为唯一不被压缩、永远注入 system prompt 的记忆层。agent 自己决定记什么，比任何自动摘要都更可靠。
